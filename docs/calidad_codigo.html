Demostración de Clean Code en Proyecto

1. Nombrado claro y consistente

  • Variables, métodos y clases descriptivos y autoexplicativos.
  • Ejemplo de código:
    • // Malo
    int x;

    // Bueno
    int numeroDeClientesActivos;
  • Clases en el proyecto: ClienteService, ClienteController, ClienteRepositoryImpl.

2. Funciones cortas y con un solo propósito

// Malo
public void procesarCliente(...) { validarDatos(...); guardarCliente(...); enviarNotificacion(...); }

// Bueno
public void procesarCliente(...) { validarDatos(...); guardarCliente(...); notificarCliente(...); }

Ejemplo real: ClienteService solo maneja lógica de negocio, ClienteController solo maneja la API.

3. Eliminación de código duplicado

Se reutilizan métodos comunes, por ejemplo:

validarDocumento(String tipo, long numero);

4. Manejo consistente de excepciones y HTTP

Se implementó GlobalExceptionHandler para centralizar 400, 404 y 500.

5. Logging estructurado

logger.info("Cliente encontrado: {}", cliente);
logger.warn("Cliente no encontrado: {}", tipoDocumento);
logger.error("Error inesperado", ex);

6. Inyección de dependencias y bajo acoplamiento

public ClienteService(ClienteRepository clienteRepository) { this.clienteRepository = clienteRepository; }

7. Testing automatizado

  • Pruebas unitarias con Mockito y de integración con MockMvc.
  • Se prueban ClienteService y ClienteController.

8. Documentación mínima y legible

Comentarios explicativos solo donde la intención del código no es evidente.

9. Estilo consistente y constantes

private static final String MSG_NO_ENCONTRADO = "Cliente no encontrado";

10. Separación de responsabilidades por capas

  • Controller → API y HTTP
  • Service → lógica de negocio
  • Repository → acceso a datos
  • Domain → modelos de negocio
Evidencia para presentar:
  • Fragmentos de código bien nombrados y documentados.
  • Commits que muestran refactorización y limpieza de código.
  • Tests unitarios y de integración aplicados correctamente.
  • Manejo consistente de logs y excepciones.
