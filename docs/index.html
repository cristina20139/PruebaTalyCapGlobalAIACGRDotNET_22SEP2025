Introducción

Este proyecto consiste en la creación de servicios REST para la consulta de información de clientes, desarrollado como parte de una prueba práctica de backend. La aplicación se implementa utilizando Java Spring Boot con Maven y se conecta a una base de datos MS SQL Server 2019, integrando buenas prácticas de arquitectura en capas y documentación de endpoints mediante Swagger.

Objetivo

Permitir la consulta de datos de clientes por su número de identificación, asegurando respuestas adecuadas mediante códigos HTTP estándar:

  • 200 OK
  • 400 Bad Request
  • 404 Not Found
  • 500 Internal Server Error

Características principales

  • Base de Datos MS SQL Server 2019: Tabla Clientes y procedimientos almacenados para obtener información.
  • Arquitectura en Capas: Separación de responsabilidades en Controllers, Services, Repositories y DTOs.
  • Servicio Mockeado: Retorna información únicamente para el cliente con número de documento 23445322.
  • Swagger: Documentación y prueba de endpoints de forma interactiva.
  • Entrega: Código fuente completo https://github.com/cristina20139/PruebaTalyCapGlobalAIACGRDotNET_22SEP2025.

Notas adicionales

Se consideraron buenas prácticas de desarrollo backend, incluyendo manejo de errores, arquitectura modular y calidad de código. Se sugiere la incorporación de logs y pruebas unitarias como valores agregados.
