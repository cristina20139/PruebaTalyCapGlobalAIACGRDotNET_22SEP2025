Pregunta 15 – Patrones de Diseño

¿Conoce patrones de diseño? De ser así, mencione dos patrones de diseño y demuestre un pequeño ejemplo. Además, mencione al menos 15 patrones genéricos y luego patrones REST categorizados.

Respuesta:

Un patrón de diseño es una solución reutilizable y probada a un problema común en el diseño de software. Facilita la mantenibilidad, escalabilidad y comprensión del código.

15 Patrones Genéricos de P.O.O:

  • 1. Singleton – Garantiza una única instancia de una clase.
  • 2. Factory Method – Crea objetos sin exponer la lógica de creación.
  • 3. Abstract Factory – Crea familias de objetos relacionados.
  • 4. Builder – Construye objetos complejos paso a paso.
  • 5. Prototype – Crea nuevos objetos clonando un prototipo.
  • 6. Adapter – Convierte la interfaz de una clase en otra interfaz esperada.
  • 7. Decorator – Agrega responsabilidades adicionales a un objeto dinámicamente.
  • 8. Proxy – Controla el acceso a un objeto.
  • 9. Facade – Proporciona una interfaz simplificada a un subsistema.
  • 10. Observer – Permite notificar cambios a múltiples dependientes.
  • 11. Strategy – Permite cambiar el comportamiento de un algoritmo en tiempo de ejecución.
  • 12. Command – Encapsula una solicitud como objeto, permitiendo operaciones parametrizadas.
  • 13. Iterator – Permite recorrer elementos de una colección sin exponer su estructura interna.
  • 14. Composite – Permite tratar objetos individuales y compuestos de manera uniforme.
  • 15. State – Permite que un objeto cambie su comportamiento cuando cambia su estado interno.

Patrones REST/API categorizados:

  • Patrones de URI: Resource Naming, Versioning, Filtering.
  • Patrones de Comportamiento: HATEOAS, Caching, Pagination.
  • Patrones de Respuesta: Error Handling, Standard Response Wrappers, Rate Limiting.
  • Patrones de Seguridad: OAuth2, JWT Authentication, Throttling.
  • Patrones de Integración: API Gateway, Circuit Breaker, Retry Pattern.

Ejemplo práctico de dos patrones:

1. Singleton (Creacional)

public class ConexionDB {
    private static ConexionDB instancia;

    private ConexionDB() {
        System.out.println("Conexión creada");
    }

    public static ConexionDB getInstancia() {
        if (instancia == null) {
            instancia = new ConexionDB();

        }
        return instancia;
    }
}

public class Main {
    public static void main(String[] args) {
        ConexionDB c1 = ConexionDB.getInstancia();
        ConexionDB c2 = ConexionDB.getInstancia();
        System.out.println(c1 == c2); // true, misma instancia
    }
}

2. Adapter (Estructural)

// Clase existente
class LectorCSV {
    public void leerCSV() {
        System.out.println("Leyendo CSV...");
    }
}

// Interfaz esperada
interface Lector {
    void leer();
}

// Adapter
class LectorCSVAdapter implements Lector {
    private LectorCSV lectorCSV;

    public LectorCSVAdapter(LectorCSV lectorCSV) {
        this.lectorCSV = lectorCSV;
    }

    @Override
    public void leer() {
        lectorCSV.leerCSV(); // Adaptando método
    }
}

public class MainAdapter {
    public static void main(String[] args) {
        Lector lector = new LectorCSVAdapter(new LectorCSV());
        lector.leer(); // Funciona usando la interfaz Lector
    }
}
Nota: Los patrones de diseño ayudan a construir software escalable, mantenible y consistente, y su aplicación en REST/API permite una arquitectura robusta y reutilizable.
