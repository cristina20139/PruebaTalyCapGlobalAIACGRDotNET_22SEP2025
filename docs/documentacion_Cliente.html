📄 Documentación de la clase Cliente

📌 Rol en la Arquitectura Hexagonal

La clase Cliente pertenece a la capa de dominio. 👉 Aquí es donde se definen las entidades puras que representan el modelo de negocio.

  • 🏗️ Núcleo del dominio: No depende de frameworks ni de la infraestructura.
  • 🧩 Entidad independiente: Puede ser usada tanto por la capa de aplicación (servicios) como por la infraestructura (repositorios, controladores).
  • 🔄 Puerto y adaptadores: Se comunica indirectamente con la base de datos o APIs a través de interfaces (repositorios), manteniendo el principio de inversión de dependencias.

🚀 Uso de Spring Boot en estos fuentes

Aunque la clase Cliente no usa directamente Spring Boot, dentro del proyecto cumple un papel clave:

  • 📦 Spring Boot facilita la inyección de dependencias para repositorios y servicios que trabajan con Cliente.
  • 🌐 Los controladores REST de Spring Boot exponen APIs que reciben/envían objetos Cliente en formato JSON.
  • 🗄️ Spring Data (si se usa) permite mapear automáticamente la entidad Cliente a tablas en la base de datos.
👉 En conclusión, Spring Boot actúa como infraestructura, pero Cliente sigue siendo puro dominio.

Descripción: Representa un cliente en el sistema. Pertenece a la capa de dominio y define la entidad Cliente con sus atributos y métodos de acceso (getters/setters).

🌟 Principios aplicados

  • ✅ SRP (Single Responsibility Principle) : la clase solo representa la información de un cliente.
  • 🔒 Encapsulamiento: atributos privados con getters y setters.
  • 🧹 Clean Code: nombres claros y consistentes.
  • 🏛️ Hexagonal: no depende de infraestructura ni frameworks.
