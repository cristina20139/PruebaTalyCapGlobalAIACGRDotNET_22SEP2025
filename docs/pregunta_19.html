Pregunta 19 – Exception vs NullPointerException

¿Qué diferencia existe al controlar una excepción de tipo Exception y una excepción NullPointerException en Java?

Respuesta:

- Exception: Es la clase base para la mayoría de excepciones comprobadas en Java. Capturar Exception permite manejar cualquier excepción derivada, pero puede ocultar errores específicos. - NullPointerException (NPE): Es una excepción no comprobada que ocurre cuando se intenta usar un objeto que tiene valor null. Capturar específicamente NPE permite manejar casos donde se accede a referencias nulas sin afectar otras excepciones.

Ejemplo en Java:

1. Controlando Exception

public class EjemploException {
    public static void main(String[] args) {
        try {
            String texto = null;
            System.out.println(texto.length()); // Lanza NullPointerException
        } catch (Exception e) {
            System.out.println("Se capturó una excepción: " + e.getClass().getSimpleName());
        }
    }
}

2. Controlando NullPointerException

public class EjemploNPE {
    public static void main(String[] args) {
        try {
            String texto = null;
            System.out.println(texto.length()); // Lanza NullPointerException
        } catch (NullPointerException e) {
            System.out.println("Se capturó un NullPointerException específicamente.");
        }
    }
}
Nota: Capturar Exception es más general y puede ocultar otros errores, mientras que capturar NullPointerException permite un manejo más preciso de referencias nulas.
