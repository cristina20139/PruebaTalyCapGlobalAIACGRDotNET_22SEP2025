Pregunta 14 – Opción Múltiple

En Java, la mejor alternativa para ejecutar una consulta de actualización, inserción y/o eliminación de datos sobre una base de datos es:

  • a) .execute();
  • b) .executeUpdate();
  • c) .execute(-consulta-);
  • d) Todas las anteriores

Respuesta Correcta:

b) .executeUpdate();

Justificación:

En Java, al usar JDBC para interactuar con una base de datos:

  • executeUpdate() se utiliza para ejecutar sentencias SQL que modifican datos (INSERT, UPDATE, DELETE) y retorna el número de filas afectadas.
  • execute() puede ejecutar cualquier tipo de sentencia SQL, pero su uso es más general y requiere manejo adicional para saber si devuelve un ResultSet o un conteo de filas.
  • La opción c) es incorrecta porque execute(-consulta-) no es una sintaxis válida.
Por lo tanto, la opción más adecuada para actualizar, insertar o eliminar registros es executeUpdate().

Nota: Para consultas SELECT se recomienda usar executeQuery(), que retorna un ResultSet.

Ejemplo en Java:

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;

public class Main {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/miBase";
        String usuario = "root";
        String password = "1234";

        try (Connection con = DriverManager.getConnection(url, usuario, password)) {
            // Consulta de inserción
            String sql = "INSERT INTO empleados (nombre, edad) VALUES (?, ?)";
            try (PreparedStatement stmt = con.prepareStatement(sql)) {
                stmt.setString(1, "Manuela");
                stmt.setInt(2, 30);

                int filasAfectadas = stmt.executeUpdate();
                System.out.println("Filas insertadas: " + filasAfectadas);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
