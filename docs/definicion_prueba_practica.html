PRACTICA

Segundo Punto

Objetivo: Cree una aplicación en Visual Studio 2022 (.NET 6/7 Web API) que consulte los datos de un cliente por su número de identificación desde una base de datos SQL Server 2019.

Requisitos

  • Base de Datos: Crear la base de datos DBClientes en SQL Server 2019 con la tabla Clientes.
  • Procedimiento Almacenado: Crear un SP para obtener los datos del cliente.
  • Estructura en Capas (Visual Studio 2022):
    • API: Controladores y endpoints
    • Services: Lógica de negocio
    • Repositories: Acceso a datos con Entity Framework Core
    • DTOs: Objetos de transferencia de datos
  • Swagger: Documentar los endpoints con Swagger.
Endpoint a Implementar:
GET /api/clientes/{identificación}
Debe devolver los datos del cliente si existe.

Tercer Punto

PRUEBA PRÁCTICA – BACKEND: Crear un servicio REST para consultar información básica de un cliente.

Condiciones del Servicio

  • Manejo de Códigos HTTP:
    • 200 OK
    • 400 Bad Request
    • 404 Not Found
    • 500 Internal Server Error
  • Entrada:
    • Tipo de documento (C = Cédula de ciudadanía, P = Pasaporte)
    • Número de documento
    • Ambos obligatorios.
  • Salida (mockeada, solo para cliente 23445322):
    • Primer nombre
    • Segundo nombre
    • Primer apellido
    • Segundo apellido
    • Teléfono
    • Dirección
    • Ciudad de residencia

Tecnologías y Entregables

  • Spring & Maven: Uso obligatorio.
  • Puerto: La aplicación debe iniciar en el puerto 8090.
  • Entregables:
    • JAR de la aplicación
    • Código fuente
Notas:
  • Los datos de respuesta son mockeados (solo retornar para cliente 23445322).
  • Valores agregados (no obligatorios): manejo de logs, pruebas unitarias y calidad de código.
