Pregunta 4 – Opción Múltiple

¿Qué diferencia existe entre una matriz de datos y un vector de datos?

  • a) La matriz permite cambios en su dimensión, algo que no ocurre con el vector
  • b) La matriz no permite ningún cambio en sus estructuras de datos, el vector permite cambio solamente en sus estructuras de datos iniciales
  • c) La matriz en la mayoría de los casos relaciona filas y columnas, mientras que el vector relaciona solamente filas
  • d) Todas las respuestas son correctas

Respuesta Correcta:

c) La matriz en la mayoría de los casos relaciona filas y columnas, mientras que el vector relaciona solamente filas

Justificación:

En Java y otros lenguajes de programación, un vector (o un array unidimensional) representa una secuencia de elementos lineales, mientras que una matriz (array bidimensional) organiza datos en filas y columnas. La opción a) es incorrecta porque tanto matrices como vectores en Java tienen tamaño fijo una vez creados. La opción b) es confusa, ya que el vector tampoco cambia su tamaño automáticamente (a menos que usemos ArrayList). Por ello, la diferencia principal y correcta es la de la opción c).

Nota: Para manejar cambios de tamaño dinámico se recomienda usar ArrayList en lugar de arrays fijos.

Ejemplo en Java:

public class Main {
    public static void main(String[] args) {
        // Vector (array unidimensional)
        int[] vector = {1, 2, 3, 4, 5};
        System.out.println("Vector:");
        for(int v : vector){
            System.out.println(v);
        }

        // Matriz (array bidimensional)
        int[][] matriz = {
            {1, 2, 3},
            {4, 5, 6},
            {7, 8, 9}
        };
        System.out.println("\nMatriz:");
        for(int i = 0; i < matriz.length; i++){
            for(int j = 0; j < matriz[i].length; j++){
                System.out.print(matriz[i][j] + " ");
            }
            System.out.println();
        }
    }
}
