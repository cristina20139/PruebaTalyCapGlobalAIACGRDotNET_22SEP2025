Documentación Detallada del Diagrama de Casos de Uso UML – Sistema Gestión de Clientes

1. Descripción General

Este documento describe los principales casos de uso del Sistema Gestión de Clientes mediante un diagrama UML de casos de uso. El diagrama identifica los actores externos (Administrador, Usuario API) y sus interacciones clave con el sistema.

2. Versión y Especificación UML

  • Versión de UML: 2.5.1
  • Tipo de diagrama: Diagrama de Casos de Uso (Use Case Diagram)
  • Año de creación: 2025
Nota: El estándar UML 2.5.1 para diagramas de casos de uso permite modelar de manera sencilla las funcionalidades principales desde la perspectiva de los usuarios/actores.

3. Explicación del Diagrama

  1. Actores:
    • Administrador: Puede generar clientes de prueba para propósitos de desarrollo o pruebas internas.
    • Usuario API: Representa a sistemas externos o usuarios que interactúan mediante la API para consultar o crear clientes, y recibir mensajes de error claros.
  2. Casos de uso principales:
    • Generar clientes de prueba: Accesible solo para el Administrador.
    • Consultar cliente por documento: Permite buscar información de un cliente usando un identificador. Disponible para el Usuario API.
    • Crear cliente: Permite registrar un nuevo cliente en el sistema mediante la API.
    • Recibir errores claros: El sistema proporciona respuestas claras y estandarizadas a los errores (400, 404, 500), facilitando la integración y depuración.
  3. Extensión: El caso de uso "Recibir errores claros" incluye explícitamente la devolución de códigos de error HTTP estándar, como se indica en la anotación.

4. Justificación del Uso de Diagrama UML de Casos de Uso

El diagrama de casos de uso es la mejor opción para:
  • Visualizar rápidamente las funcionalidades principales del sistema desde la perspectiva de los usuarios y sistemas externos.
  • Facilitar la comunicación con stakeholders no técnicos y equipos de desarrollo.
  • Definir el alcance funcional y los requisitos iniciales de la solución.
  • Establecer límites claros sobre qué actores pueden acceder a cada funcionalidad.

5. Estilo Visual

El documento utiliza un estilo limpio, legible y profesional:
  • Colores suaves y bloques diferenciados para resaltar notas, advertencias y ejemplos.
  • Tipografía moderna y clara.
  • El diagrama se muestra centrado, con borde y anotaciones para facilitar el entendimiento visual.
