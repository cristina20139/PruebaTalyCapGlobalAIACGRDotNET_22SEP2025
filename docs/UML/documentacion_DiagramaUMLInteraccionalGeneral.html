Documentación Detallada del Diagrama de Secuencia UML – Escenarios de Cliente

1. Descripción General

Este documento describe, mediante un diagrama de secuencia UML avanzado, los principales escenarios de interacción del usuario con el sistema de gestión de clientes: consulta, creación y generación de clientes de prueba. Se visualizan actores, componentes y el flujo de mensajes con manejo de condiciones y respuestas HTTP.

2. Versión y Especificación UML

  • Versión de UML: 2.5.1
  • Tipo de diagrama: Diagrama de Secuencia (Sequence Diagram, con fragmentos alternativos)
  • Año de creación: 2025
Nota: El diagrama aplica fragmentos alt para ramificación (cliente existe/no existe) y muestra tres escenarios clave con sus flujos y respuestas.

3. Explicación del Diagrama

  1. Escenario: Consultar Cliente
    • El usuario solicita un cliente vía GET /clientes/{tipo}/{documento}.
    • ClienteController delega la consulta a ClienteService, que a su vez consulta en ClienteRepositoryImpl.
    • Fragmento alt: Si el cliente existe, retorna 200 OK + Cliente; si no existe, se lanza ClienteNoEncontradoException y retorna 404 Not Found vía GlobalExceptionHandler.
  2. Escenario: Crear Cliente
    • El usuario envía un POST /clientes con los datos del cliente.
    • La creación fluye desde ClienteController a ClienteService y luego a ClienteRepositoryImpl.
    • Al crearse correctamente, se responde con 201 Created + Cliente.
  3. Escenario: Generar Clientes de Prueba
    • El usuario solicita generación masiva con generarClientesPrueba(cantidad).
    • ClienteGenerator crea y guarda los clientes de prueba, retornando la lista generada al usuario.

4. Justificación del Uso de Diagrama UML de Secuencia Avanzada

Este diagrama es ideal porque:
  • Permite visualizar flujos complejos y múltiples escenarios en un solo modelo.
  • Fragmentos alternativos alt facilitan el entendimiento de condiciones (éxito/fracaso).
  • Documenta el flujo de interacciones y el manejo de excepciones/respuestas HTTP.
  • Es un estándar útil para desarrolladores, testers y analistas funcionales.

5. Estilo Visual

El documento mantiene un estilo moderno y profesional:
  • Colores suaves, bloques diferenciados y tipografía clara.
  • El diagrama está centrado y segmentado por escenarios para fácil navegación.
  • Se destacan actores, componentes y ramificaciones condicionales.
