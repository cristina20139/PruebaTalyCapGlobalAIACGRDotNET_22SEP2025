Documentación Detallada del Diagrama de Despliegue UML – Sistema Gestión de Clientes

1. Descripción General

Este documento describe la infraestructura de despliegue del Sistema Gestión de Clientes mediante un diagrama de despliegue UML. Se ilustran los principales nodos físicos y virtuales, así como la comunicación y ubicación de los componentes de software en la arquitectura.

2. Versión y Especificación UML

  • Versión de UML: 2.5.1
  • Tipo de diagrama: Diagrama de Despliegue (Deployment Diagram)
  • Año de creación: 2025
Nota: El diagrama de despliegue UML 2.5.1 permite modelar la topología física y lógica del sistema, ubicando componentes en hardware y especificando protocolos de comunicación.

3. Explicación del Diagrama

  1. Usuario & Internet: El usuario accede al sistema a través de Internet, utilizando el protocolo seguro HTTPS.
  2. Servidor Web: El tráfico llega primero a un servidor web (Nginx/Apache) que actúa como proxy inverso y balanceador, recibiendo peticiones HTTPS y reenviándolas como HTTP al servidor de aplicaciones.
  3. Servidor de Aplicaciones: Contiene el backend implementado en Spring Boot, con componentes como ClienteController, ClienteService y GlobalExceptionHandler.
  4. Servidor de Base de Datos: Alberga PostgreSQL, accedido vía JDBC desde el servidor de aplicaciones para persistencia de datos.
  5. Servidor Utilidades: Proporciona servicios auxiliares, como ClienteGenerator, que puede ser consumido vía REST/HTTP.
  6. Comunicación:
    • HTTPS entre Usuario–Internet–Servidor Web.
    • HTTP (proxy) entre Servidor Web y Servidor de Aplicaciones.
    • JDBC entre Servidor de Aplicaciones y Base de Datos.
    • REST/HTTP entre Servidor de Aplicaciones y Servidor Utilidades.

4. Justificación del Uso de Diagrama UML de Despliegue

El diagrama de despliegue UML es fundamental porque:
  • Visualiza la infraestructura física y lógica del sistema.
  • Permite planificar la seguridad, escalabilidad y disponibilidad.
  • Deja claro el flujo de comunicación y los protocolos utilizados (HTTPS, HTTP, JDBC, REST).
  • Facilita la comunicación entre equipos de desarrollo, operaciones y arquitectura.

5. Estilo Visual

El documento mantiene un estilo profesional y homogéneo:
  • Colores suaves, tipografía moderna y bloques diferenciados.
  • El diagrama está centrado, con bordes y etiquetas para facilitar la comprensión.
  • Los nodos y componentes están claramente identificados y relacionados realísticamente.
