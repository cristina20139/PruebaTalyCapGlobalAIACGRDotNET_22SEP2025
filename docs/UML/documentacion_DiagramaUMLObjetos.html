Documentación Detallada del Diagrama de Objetos UML – Escenario Consulta de Cliente

1. Descripción General

Este documento describe cómo se relacionan los objetos principales en tiempo de ejecución durante el proceso de consulta de cliente, usando un diagrama de objetos UML. El diagrama muestra referencias, relaciones y valores de atributos de un ejemplo real de consulta.

2. Versión y Especificación UML

  • Versión de UML: 2.5.1
  • Tipo de diagrama: Diagrama de Objetos (Object Diagram)
  • Año de creación: 2025
Nota: UML 2.5.1 permite modelar instancias concretas y relaciones entre objetos en tiempo de ejecución, ilustrando cómo se materializan las asociaciones y dependencias en un caso concreto.

3. Explicación del Diagrama

  1. ClienteController contiene una referencia (clienteService = service) al servicio de clientes.
  2. ClienteService contiene una referencia (clienteRepository = repository) al repositorio de clientes.
  3. ClienteRepositoryImpl es la implementación concreta del repositorio consultado.
  4. En una consulta, ClienteService utiliza ClienteRepositoryImpl y retorna un objeto Cliente al controlador.
  5. El objeto Cliente mostrado es una instancia real con valores de atributos, ejemplificando el resultado devuelto al usuario:
    • tipoDocumento: 'C'
    • numeroDocumento: 12345678
    • primerNombre: 'Sofía'
    • segundoNombre: 'Lucía'
    • primerApellido: 'Gómez'
    • segundoApellido: 'Martínez'
    • telefono: '300-1234567'
    • direccion: 'Calle 45 # 12-10'
    • ciudadResidencia: 'Bogotá'

4. Justificación del Uso de Diagrama UML de Objetos

El diagrama de objetos UML es útil porque:
  • Permite visualizar instancias reales y relaciones de objetos en un momento específico de la ejecución.
  • Facilita la comprensión de la estructura en tiempo de ejecución y los valores concretos retornados al usuario.
  • Es ideal para validar el diseño y la implementación, así como para documentación y pruebas.
  • Complementa otros diagramas estructurales mostrando ejemplos concretos.

5. Estilo Visual

El documento mantiene un estilo profesional, uniforme y claro:
  • Colores suaves, bloques y tipografía moderna.
  • El diagrama está centrado, con relaciones y valores de atributos claramente visibles.
  • Los nombres de objetos y relaciones están alineados con la nomenclatura de la aplicación real.
